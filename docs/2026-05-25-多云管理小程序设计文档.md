# 多云管理小程序设计文档

**项目名称**：MultiCloud Manager MiniProgram  
**目标用户**：3-10 人小团队，需要统一管理多个云平台资源  
**核心功能**：跨云资源查看、控制、新建，支持 Terraform 模板管理，AI Agent 智能编排

### 1.2 技术栈

| 组件 | 技术选型 | 说明 |
|---|---|---|
| 前端 | 微信原生小程序 + Vant Weapp UI | 支持微信生态，无需安装 |
| AI Agent 层 | LLM API + 规则引擎 | 意图解析、方案生成、风险评估 |
| 凭证代理层 | Infisical Agent Vault | 凭证代理，AI Agent 永不接触真实密钥 |
| 后端 | Go 1.21+ + Gin 框架 | 高性能，与 Cloudpods SDK 同语言 |
| 数据库 | PostgreSQL (Render 托管) | 关系型，支持 JSONB |
| 缓存 | Redis (Render Redis) | API 响应缓存 |
| 多云 SDK | Cloudpods `pkg/cloudprovider` | 复用成熟的多云适配代码 |
| 部署 | Render Web Service (免费 tier) | 统一在云体系内 |
| Terraform 执行 | Docker 容器 + Terraform CLI | 沙箱环境执行用户模板 |

### 1.3 AI Agent 编排层设计

#### 1.3.1 AI Agent 职责

AI Agent 层是小程序的"智能大脑"，用户通过自然语言与 Agent 交互，Agent 自动拆解意图、规划执行路径、评估风险并调用后端 API。

**核心能力**：
- **意图解析**：将自然语言（"帮我在 Azure 新加坡区开一台 2核4G 的 VM"）转为结构化指令
- **方案生成**：根据多轮对话上下文，自动补全缺失参数（规格、网络、存储等），生成可执行方案
- **风险审查**：在敏感操作前自动评估风险级别，标记需要人工确认的步骤
- **多步编排**：将复杂任务（如"迁移 Oracle 上的数据库到 Azure"）拆解为有序执行序列

#### 1.3.2 四步执行流水线

```
用户输入
   │
   ▼
Step 1: 意图解析 (Intent Parsing)
   │  LLM 识别操作类型、目标云、资源类型、参数
   │  输出：结构化 Intent 对象
   ▼
Step 2: 方案生成 (Plan Generation)
   │  查询云账户信息、补全缺失参数、生成执行计划
   │  输出：执行 Plan（含步骤序列 + 预估影响）
   ▼
Step 3: 风险审查 (Risk Review)
   │  逐步骤分析风险级别（🟢 低 / 🟡 中 / 🔴 高）
   │  高风险步骤 → 必须用户确认；中风险 → 展示影响摘要
   │  输出：Plan + 风险标注
   ▼
Step 4: 执行与反馈 (Execution)
   │  调用后端 API 执行，实时反馈进度
   │  输出：每一步的执行结果
```

#### 1.3.4 执行模式

| 模式 | 描述 | 适用场景 |
|---|---|---|
| **仅生成方案** | AI 返回 plan，用户自行在其他渠道执行 | 需要审批、学习型用户 |
| **分步确认** | 每一步执行前等待确认 | 涉及生产环境操作 |
| **风险审查模式** | 自动执行低风险步骤，高风险步骤暂停确认 | 混合场景 |
| **全自动执行** | 用户一次性确认整体方案后自动执行 | 开发/测试环境 |

#### 1.3.5 Agent 安全约束（硬编码规则引擎，不依赖 LLM）

以下规则由规则引擎强制执行，**LLM 无法绕过**：

| 规则 | 描述 |
|---|---|
| 禁止批量删除 | 一次 plan 中最多删除 5 个资源，超出需分批确认 |
| 禁止清空存储桶 | `delete_bucket` 操作必须用户明确二次确认 |
| 禁止修改 IAM 角色 | AI 可以建议，但不能写入 plan |
| 禁止修改计费相关设置 | 如切换付费模式、调整预算告警 |
| 操作审计 | 所有 AI 生成的 plan 和执行记录写入 `ai_agent_logs` |

### 1.4 Agent Vault 凭证代理层设计

#### 1.4.1 设计目标

AI Agent 层是第三方 LLM 服务，**绝对不能接触到真实的云平台 API 密钥**。Agent Vault 作为凭证代理，确保：

- AI Agent 的请求只包含 `credential_ref`（如 `"azure-prod-account"`）
- 真实密钥（`AZURE_CLIENT_SECRET`、`OCI_PRIVATE_KEY` 等）仅存在于 Agent Vault 中
- 后端 API 在转发请求前，由 Agent Vault 注入真实凭据并**立即清除临时注入**

#### 1.4.2 数据流

```
AI Agent 生成请求                 Agent Vault 注入                  多云适配层接收
┌─────────────────┐            ┌─────────────────┐            ┌─────────────────┐
│ {               │            │ 查询 credential_ │            │ {               │
│   "action": "..",│  ──────▶  │ ref → 读取真实    │  ──────▶  │   "action": "..",│
│   "credential_  │            │ 密钥 → 注入请求   │            │   "azure_client_│
│    ref": "azure- │            │ → 签名后转发     │            │    secret": "真  │
│    prod"         │            │ → 立即丢弃请求   │            │    实密钥"       │
│ }               │            └─────────────────┘            │ }               │
└─────────────────┘                                           └─────────────────┘
       ▲                                                            │
       │ 只看占位符                                                  │ 不感知 Agent Vault
       │                                                            ▼
   AI Agent                                                   Cloud API 调用
  (第三方 LLM)
```

#### 1.4.3 Agent Vault 部署

Agent Vault 与后端 API 部署在同一个 Render 私有网络中，**不暴露公网端口**。

```yaml
# render.yaml (追加)
  - type: web
    name: agent-vault
    env: docker
    dockerfilePath: ./vault/Dockerfile
    envVars:
      - key: VAULT_UNSEAL_KEY
        generateValue: true
      - key: VAULT_MASTER_TOKEN
        generateValue: true
      - key: BACKEND_API_URL
        value: "http://multicloud-backend:8080"
    # 内部网络，不暴露公网
```

#### 1.4.4 凭证注入流程

```
1. 后端 API 收到 AI Agent 发来的执行请求
      │
2. 后端 API 发现 credential_ref，改写请求 URL 指向 Agent Vault
      │  POST http://agent-vault:8200/api/v1/inject
      │  Body: {"credential_ref": "azure-prod", "action_request": {...}}
      ▼
3. Agent Vault 从安全存储读取真实密钥
      │  密钥来源：Render 环境变量 / Vault KV Store
      ▼
4. Agent Vault 将密钥注入 action_request，签名
      │  → POST https://management.azure.com/... (含真实凭据)
      ▼
5. 返回原始响应给后端 API，后端 API 返回给前端
      │  密钥在内存中仅存留 1 次请求周期
```

#### 1.4.5 密钥存储策略

| 密钥类型 | 存储方式 | 轮换策略 |
|---|---|---|
| 主加密密钥 | Render 环境变量（加密），不在任何配置文件中 | 每 90 天提示轮换 |
| 云平台 API 密钥 | Agent Vault KV Store，AES-256-GCM 加密 | 与云平台官方建议一致 |
| 微信 AppSecret | Render 环境变量 | 按微信要求 |
| JWT 签名密钥 | Agent Vault，启动时注入 | 每 30 天自动轮换 |

#### 1.4.6 AI Agent 请求格式

所有 AI Agent 发出的请求中，凭据字段**必须使用引用格式**，而非明文：

```json
// ✅ 正确：AI Agent 发出的请求
{
    "action": "list_vm",
    "cloud": "azure",
    "credential_ref": "azure-prod-main"
}

// ❌ 禁止：AI Agent 不得在请求中包含任何真实密钥
{
    "action": "list_vm",
    "cloud": "azure",
    "client_id": "xxxxx",       // ← 禁止
    "client_secret": "yyyyy"    // ← 禁止
}
```

#### 1.4.7 审计与告警

- Agent Vault 所有密钥访问记录到 `vault_audit_log`
- 异常检测：同一密钥 1 分钟内访问超过 10 次 → 触发告警，临时冻结该密钥
- AI Agent 尝试在请求中包含非 `credential_ref` 的凭据类字段 → 请求被后端拦截，记录安全事件

## 2. 数据模型

### 2.1 核心表设计

#### `users` 用户表
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    openid VARCHAR(128) UNIQUE NOT NULL,      -- 微信 OpenID
    nickname VARCHAR(100),                     -- 微信昵称
    avatar_url TEXT,                           -- 头像 URL
    team_id UUID REFERENCES teams(id),         -- 所属团队
    role VARCHAR(20) DEFAULT 'member',         -- 角色: admin/member
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `teams` 团队表
```sql
CREATE TABLE teams (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,                -- 团队名称
    description TEXT,
    created_by UUID REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `cloud_accounts` 云账户表
```sql
CREATE TABLE cloud_accounts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    team_id UUID REFERENCES teams(id) NOT NULL,
    cloud_type VARCHAR(20) NOT NULL,           -- azure/oracle/render/tencent
    name VARCHAR(100) NOT NULL,                 -- 账户别名
    encrypted_credentials BYTEA NOT NULL,       -- AES-GCM 加密的凭据
    encryption_key_id VARCHAR(64),              -- 密钥版本
    is_active BOOLEAN DEFAULT true,
    last_sync_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `resources_cache` 资源缓存表
```sql
CREATE TABLE resources_cache (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    account_id UUID REFERENCES cloud_accounts(id),
    resource_type VARCHAR(50) NOT NULL,         -- vm/db/storage
    cloud_resource_id VARCHAR(255) NOT NULL,    -- 云平台原始 ID
    cloud_region VARCHAR(50),                   -- 区域
    name VARCHAR(200),                          -- 资源名称
    status VARCHAR(50),                         -- 运行状态
    spec JSONB,                                 -- 规格详情
    tags JSONB,                                 -- 标签
    last_synced_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(account_id, cloud_resource_id)
);
```

#### `terraform_templates` Terraform 模板表
```sql
CREATE TABLE terraform_templates (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    team_id UUID REFERENCES teams(id) NOT NULL,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    content TEXT NOT NULL,                      -- .tf 文件内容
    variables JSONB,                            -- 变量定义
    cloud_account_ids UUID[],                   -- 关联的云账户
    version INTEGER DEFAULT 1,
    created_by UUID REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `terraform_runs` Terraform 执行记录表
```sql
CREATE TABLE terraform_runs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    template_id UUID REFERENCES terraform_templates(id),
    run_type VARCHAR(20) NOT NULL,              -- plan/apply/destroy
    status VARCHAR(20) NOT NULL,                -- pending/running/success/failed
    plan_output TEXT,                           -- plan 输出
    apply_output TEXT,                          -- apply 输出
    variables JSONB,                            -- 本次执行变量值
    approved_by UUID REFERENCES users(id),      -- 审批人（如需）
    started_at TIMESTAMP,
    completed_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `ai_agent_sessions` AI Agent 会话表
```sql
CREATE TABLE ai_agent_sessions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) NOT NULL,
    team_id UUID REFERENCES teams(id) NOT NULL,
    session_id VARCHAR(64) UNIQUE NOT NULL,     -- 前端使用的会话 ID
    title VARCHAR(200),                         -- 会话标题（自动生成）
    status VARCHAR(20) DEFAULT 'active',        -- active/archived
    last_message_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `ai_agent_messages` AI Agent 消息表
```sql
CREATE TABLE ai_agent_messages (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID REFERENCES ai_agent_sessions(id) NOT NULL,
    role VARCHAR(10) NOT NULL,                  -- user/assistant/system
    content TEXT NOT NULL,                      -- 消息内容
    metadata JSONB,                             -- 结构化数据（intent/plan/risk）
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `ai_agent_plans` AI Agent 执行计划表
```sql
CREATE TABLE ai_agent_plans (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID REFERENCES ai_agent_sessions(id) NOT NULL,
    plan_id VARCHAR(64) UNIQUE NOT NULL,
    title VARCHAR(200) NOT NULL,                -- 计划标题
    steps JSONB NOT NULL,                       -- 步骤数组
    risk_summary JSONB,                         -- 风险分析结果
    missing_params JSONB,                       -- 缺失参数列表
    estimated_cost DECIMAL(10,2),               -- 预估成本
    status VARCHAR(20) DEFAULT 'pending',       -- pending/confirmed/executing/completed/failed
    confirmed_by UUID REFERENCES users(id),
    confirmed_at TIMESTAMP,
    execution_started_at TIMESTAMP,
    execution_completed_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `ai_agent_executions` AI Agent 执行记录表
```sql
CREATE TABLE ai_agent_executions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    plan_id UUID REFERENCES ai_agent_plans(id) NOT NULL,
    step_index INTEGER NOT NULL,                -- 步骤序号
    step_data JSONB NOT NULL,                   -- 步骤详情
    status VARCHAR(20) NOT NULL,                -- pending/running/success/failed
    api_request JSONB,                          -- 实际 API 请求（含 credential_ref）
    api_response JSONB,                         -- API 响应
    error_message TEXT,
    started_at TIMESTAMP,
    completed_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `vault_audit_log` Agent Vault 审计日志表
```sql
CREATE TABLE vault_audit_log (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    credential_ref VARCHAR(100) NOT NULL,       -- 引用的凭据标识
    action VARCHAR(50) NOT NULL,                -- inject/read/rotate/delete
    request_source INET,                        -- 请求来源 IP
    user_id UUID REFERENCES users(id),          -- 关联用户
    success BOOLEAN NOT NULL,
    error_message TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `operation_logs` 操作审计表
```sql
CREATE TABLE operation_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    team_id UUID REFERENCES teams(id),
    action VARCHAR(50) NOT NULL,                -- 操作类型
    resource_type VARCHAR(50),                  -- 资源类型
    resource_id VARCHAR(255),                   -- 资源 ID
    details JSONB,                              -- 操作详情
    ip_address INET,
    user_agent TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 2.2 加密方案

### 5.1 架构
```
Terraform 执行器 (Docker 容器)
├── 模板存储卷 (只读)
├── 状态存储 (PostgreSQL/S3)
```

### 5.2 执行流程
```go
type TerraformExecutor struct {
    workdir     string          // 临时工作目录

    docker      DockerClient    // Docker 客户端
}

func (e *TerraformExecutor) ExecutePlan(
    ctx context.Context,
    templateID string,
    variables map[string]string,
) (PlanResult, error) {

    // 2. 注入云凭据到环境变量
    // 3. 启动 Docker 容器执行 `terraform plan`
    // 4. 解析输出，返回资源变更列表
```

### 5.3 安全措施
1. **网络限制**：容器无外网访问，仅允许访问云 API 端点
2. **文件系统限制**：只读挂载模板目录，无宿主机访问
3. **资源限制**：CPU/内存限制，防止 DoS
4. **凭据隔离**：每个执行使用临时凭据，执行后销毁
5. **模板验证**：解析模板，限制可创建的资源类型

## 9. 风险与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|---|---|---|---|
| **AI Agent 凭证泄露** | 高 | 极高 | Agent Vault 强制隔离，AI 只看到占位符，真实密钥仅存在于 Vault 内存 |
| **LLM 幻觉导致误操作** | 中 | 高 | 规则引擎硬编码约束（禁止批量删除、禁止修改 IAM），所有操作必须经过风险审查 |
| **Agent Vault 单点故障** | 中 | 高 | 高可用部署（Render 多实例），定期备份密钥，快速恢复机制 |
| **凭证注入延迟** | 中 | 中 | 优化 Vault 内存缓存，预加载常用凭证，监控注入延迟 |
| **Cloudpods SDK 依赖复杂** | 中 | 高 | 只提取必要包，编写适配层 |
| **Oracle Cloud API 不稳定** | 中 | 中 | 降级处理，优先其他云 |
| **Terraform 执行安全风险** | 高 | 高 | Docker 沙箱 + 资源限制 |
| **Render 免费 tier 不足** | 低 | 中 | 监控资源使用，优化查询 |
| **微信小程序审核不通过** | 中 | 高 | 提前沟通，准备合规材料，说明 AI Agent 为辅助工具 |
| **多云 API 调用频率限制** | 高 | 中 | 缓存 + 队列 + 限流 |
