# AI Assistant Mode System Design

**日期：** 2026-06-20  
**项目：** CloudOps AI 多云管理系统  
**组件：** React AI 对话页模式系统

---

## 1. 目标与背景

### 1.1 目标

将 React AI 对话页的默认同意模式从简单的"自动同意"开关改为三模式系统，模仿 OpenCode 和 Claude Code 的模式切换机制。用户可以通过分段控件在三种模式间切换，每种模式定义不同的工具执行权限。

### 1.2 核心需求

- **三种操作模式**：Plan（规划）、Action（行动）、Confirm（确认）
- **清晰的UI呈现**：使用分段控件（Segmented Control）而非简单开关
- **模式行为明确**：每种模式定义明确的工具执行规则
- **向后兼容**：替换现有的 `autoApproveMode` 布尔值
- **用户体验**：模式切换直观，当前模式始终可见

### 1.3 技术决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 模式数量 | 3种 | 覆盖只读、自动执行、手动确认三种典型场景 |
| 模式命名 | Plan/Action/Confirm | 符合AI助手领域术语，中英文对应自然 |
| UI模式 | 分段控件 | 所有选项可见，无需记忆，当前选择清晰 |
| 状态管理 | 扩展现有 chat store | 与现有架构一致，避免引入新状态管理 |

## 2. 模式定义

### 2.1 Plan Mode (规划模式)

**行为**：只读分析。允许执行非修改性工具，禁止任何写入、执行或修改操作。

**允许的工具类型**：
- 文件读取：`list_files`, `read_file`, `search_code`
- 数据分析：`analyze_data`, `generate_report`
- 查询操作：`search`, `find`, `query`
- 只读API调用：`GET` 请求

**禁止的工具类型**：
- 文件写入：`write_file`, `create_file`, `delete_file`
- 代码执行：`execute`, `run`, `build`
- 状态修改：`update`, `modify`, `change`
- 危险操作：任何 `dangerous` 级别工具

**处理非只读工具**：
- 自动拒绝非只读工具调用
- 在聊天界面显示提示："此工具在Plan模式下不可用，请切换到Action或Confirm模式"
- 不中断AI对话流程，AI可以继续生成文本响应

**使用场景**：
- 初始探索和分析
- 代码审查和理解
- 数据查询和报告生成
- 需要避免意外修改的场景

### 2.2 Action Mode (行动模式)

**行为**：自动执行。AI 可以自动执行所有工具，无需用户确认。

**允许的工具类型**：
- 所有工具类型，包括写入、执行、修改
- 无论 `dangerLevel` 如何，均自动执行

**安全考虑**：
- 用户完全信任 AI 的决策
- 类似当前的自动同意模式，但扩展到所有级别
- 适合熟悉 AI 能力且愿意承担风险的用户

**使用场景**：
- 快速原型开发
- 自动化任务执行
- 信任 AI 决策的高级用户
- 批量操作场景

### 2.3 Confirm Mode (确认模式)

**行为**：手动批准。所有工具执行都需要用户明确确认。

**确认规则**：
- 所有工具调用都显示审批弹窗
- 包括 `moderate` 和 `dangerous` 级别
- 用户必须点击"允许"或"拒绝"

**安全优势**：
- 最大程度的用户控制
- 可以审查每个工具调用的参数
- 防止意外或恶意操作

**使用场景**：
- 生产环境操作
- 敏感数据处理
- 需要审计跟踪的场景
- 新用户学习阶段

## 3. UI 设计

### 3.1 模式选择器组件

**位置**：聊天输入区域上方，固定显示

**布局**：
```
┌─────────────────────────────────────────────────┐
│ [Plan] [Action] [Confirm]    Current: Plan mode │
├─────────────────────────────────────────────────┤
│ 聊天输入框...                                   │
│ 🔒 Plan mode: Only read-only tools will be...  │
└─────────────────────────────────────────────────┘
```

**交互**：
- 分段控件，点击切换模式
- 当前模式高亮显示
- 模式切换立即生效

### 3.2 视觉设计

**颜色编码**：
- Plan：蓝色（安全、只读）
- Action：绿色（执行、行动）
- Confirm：橙色（谨慎、确认）

**状态指示**：
- 当前模式显示在选择器右侧
- 输入框下方显示模式提示文本
- 危险操作时根据模式显示不同警告

### 3.3 响应式设计

**桌面端**：分段控件水平排列  
**移动端**：分段控件可折叠为下拉菜单

## 4. 状态管理

### 4.1 扩展 Chat Store

**当前状态**：
```typescript
// 现有代码
const [autoApproveMode, setAutoApproveMode] = useState(false);
```

**新状态**：
```typescript
// 新设计
type Mode = 'plan' | 'action' | 'confirm';
const [mode, setMode] = useState<Mode>('plan');
```

**状态持久化**：
- 选择保存到 localStorage
- 跨会话保持用户偏好
- 新会话默认使用上次选择的模式

### 4.2 状态流

```
用户点击模式选择器 → 更新 mode 状态
                 → 保存到 localStorage
                 → 更新 UI 显示
                 → 影响后续工具执行审批
```

## 5. 与现有代码集成

### 5.1 需要修改的文件

1. **`ApprovalPrompt.tsx`**：
   - 移除 `autoApproveMode` 状态
   - 添加 `mode` 状态
   - 替换自动同意按钮为模式选择器
   - 更新工具执行逻辑

2. **`useExecApproval.ts`**：
   - 添加模式参数到审批逻辑
   - 根据模式决定是否自动批准

3. **聊天输入组件**：
   - 添加模式选择器UI
   - 显示当前模式提示

### 5.2 工具执行逻辑

**Plan 模式**：
```typescript
if (mode === 'plan') {
  // 只允许只读工具
  if (isReadOnlyTool(toolName)) {
    // 自动批准
  } else {
    // 拒绝或提示用户切换模式
  }
}
```

**Action 模式**：
```typescript
if (mode === 'action') {
  // 自动批准所有工具
  approveToolCall(toolCall);
}
```

**Confirm 模式**：
```typescript
if (mode === 'confirm') {
  // 总是需要确认
  showApprovalDialog(toolCall);
}
```

## 6. 用户体验流程

### 6.1 模式切换场景

**Plan → Action**：
- 用户点击 Action 模式
- UI 立即更新
- 后续工具调用自动执行
- 当前进行中的工具调用不受影响

**Action → Confirm**：
- 用户点击 Confirm 模式
- UI 立即更新
- 后续工具调用需要确认
- 已自动执行的工具不会回滚

**Confirm → Plan**：
- 用户点击 Plan 模式
- UI 立即更新
- 后续只读工具自动执行，其他工具被拒绝
- 待确认的工具调用保持待确认状态

### 6.2 边界情况处理

**模式切换期间的工具调用**：
- 模式切换是即时的
- 正在执行的工具调用继续完成
- 新工具调用使用新模式

**并发工具调用**：
- 每个工具调用独立处理
- 根据调用时的模式决定是否需要确认

## 7. 设计问题与决策

### 7.1 已决策

1. **模式命名**：Plan/Action/Confirm
2. **UI模式**：分段控件（Segmented Control）
3. **默认模式**：Plan（安全优先）
4. **状态持久化**：保存到 localStorage
5. **工具分类**：基于工具名称和dangerLevel自动分类
   - 只读工具：名称包含 `list`, `get`, `search`, `find`, `query`, `read`, `analyze` 的工具
   - 非只读工具：其他所有工具，或 `dangerLevel` 为 `moderate` 或 `dangerous` 的工具
6. **模式切换警告**：不需要警告。模式切换是即时的，用户应清楚自己的操作。
7. **模式指示器**：不在消息流中显示每个工具调用使用的模式。保持界面简洁。

## 9. 总结

本设计将简单的自动同意开关升级为三模式系统，提供更精细的工具执行控制。通过清晰的命名、直观的UI和明确的行为定义，用户可以根据场景选择最合适的模式，平衡效率与安全性。
