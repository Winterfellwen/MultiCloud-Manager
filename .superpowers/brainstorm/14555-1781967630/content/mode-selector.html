Mode Selector UI Patterns

How should users switch between Plan/Action/Confirm modes?

Option A: Toggle Button

Single button that cycles through modes on click:

📋 Plan (click to cycle)

Pros

  • Minimal UI footprint
  • Quick to click
  • Works well in tight spaces

Cons

  • Hard to see all options at once
  • Need to remember cycle order
  • Accidental clicks possible
Option B: Segmented Control

All modes visible, one selected:

Plan
Action
Confirm

Pros

  • All options visible at once
  • Clear current selection
  • No memory needed

Cons

  • Takes more space
  • More clicks to reach distant option

Recommendation

Segmented Control (Option B) is better because:

  • Users can see all available modes instantly
  • No cognitive load to remember what comes next
  • Current mode is always clear
  • Matches modern UI patterns (iOS segmented controls, Material chips)
