AI Assistant Mode System Design

Replacing auto-approve toggle with Plan/Action/Confirm mode selector

1. Mode Definitions

P

Plan Mode (规划)

Behavior: Read-only analysis. Allows list, search, and other non-mutating tools.

Example tools: list_files, search_code, read_file, analyze_data

Safety: No writes, no executions, no modifications

A

Action Mode (行动)

Behavior: Automatic execution. AI executes tools without confirmation.

Example tools: All tools including write, execute, modify

Safety: User trusts AI to make changes (like current auto-approve)

C

Confirm Mode (确认)

Behavior: Manual approval required for all tool executions.

Example tools: All tools require user confirmation

Safety: Maximum control, user approves every action

3. State Management

Implementation Approach

  • Add mode: 'plan' | 'action' | 'confirm' to chat store
  • Replace autoApproveMode boolean with mode state
  • Mode selector component reads/writes to store
  • Backend approval logic checks mode before requiring approval

Key Changes

  • Remove autoApproveMode toggle button
  • Add mode selector segmented control
  • Update approval logic to respect mode
  • Add mode indicator in chat input area

4. User Experience Flow

Mode Switching Behavior

Plan → Action

AI can now execute tools. User should be warned if in middle of conversation.

Action → Confirm

All future tools require approval. Already-executed tools are not affected.

Confirm → Plan

Return to read-only. Pending approvals remain pending.
